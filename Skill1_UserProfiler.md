# Skill: UserProfiler (Qclaw-01)

## 1. Skill 职责
解析用户提供的简历/作品集，建立结构化的人才画像，并根据“意愿优先”原则匹配目标企业和岗位。

## 2. 触发场景
- [cite_start]**意图唤醒**：当用户输入包含“投递简历”、“找工作”、“帮我投一下”、“看看有哪些岗位”、“匹配一下”等关键词时触发。 [cite: 3]
- [cite_start]**文件驱动**：用户直接上传 PDF 格式的简历或作品集文件时触发。 [cite: 9]

## 3. 执行步骤
1. [cite_start]**身份识别**：确认当前用户是否已有画像。若无，启动“宁多勿少”模式，提示用户提供简历或填写基础信息。 [cite: 9]
2. [cite_start]**结构化提取**：使用脚本解析文档，提取：基本信息、学历专业、项目/实习经历、技能点、联系方式。 [cite: 3, 5, 9]
3. [cite_start]**意愿对齐**：询问用户“第一意愿岗位”。若用户未提，则根据专业背景提供至少 3 个高匹配方向。 [cite: 8]
4. [cite_start]**生成白名单**：根据 `assets/company_list.json` 生成当前匹配的《建议投递名单》。 [cite: 5]

## 4. 输出标准
- 格式：标准的 JSON 数据包。
- [cite_start]确认机制：输出画像摘要，必须由用户回复“确认”或“修改”后方可结束。 [cite: 9]
